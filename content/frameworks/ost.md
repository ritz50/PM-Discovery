# Opportunity Solution Tree (OST)

Teresa Torres' framework connecting outcomes to experiments.

## Structure

```
Desired Outcome (business/user goal)
├── Opportunity 1 (unmet need, pain, desire)
│   ├── Solution 1a
│   │   └── Experiment 1a-i
│   └── Solution 1b
│       └── Experiment 1b-i
└── Opportunity 2
    └── ...
```

## Layers

1. **Outcome** — Measurable goal (not a feature)
2. **Opportunity** — Customer need or pain uncovered in research
3. **Solution** — Possible way to address the opportunity
4. **Experiment** — Fast test to validate a solution

## Rules

- One outcome per tree
- Opportunities come from research, not brainstorming
- Multiple solutions per opportunity
- Small experiments before big builds

## Continuous discovery

Run weekly: interview → map opportunities → ideate solutions → test.
