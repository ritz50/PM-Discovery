# PRD & Requirements

A PRD translates discovery evidence into buildable requirements.

## PRD sections

1. Problem & context (with discovery evidence)
2. Goals and non-goals
3. User stories
4. Requirements (functional + non-functional)
5. Success metrics
6. Open questions and risks

## User story format

> As a [user], I want to [action], so that [outcome].

## Acceptance criteria

Testable conditions that define "done" for each story.

## Key takeaway

A PRD without discovery evidence is a spec for a guess. Link every requirement to a validated insight.
