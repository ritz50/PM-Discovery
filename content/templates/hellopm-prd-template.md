# helloPM PRD Template

*From `content/inbox/Product Strategy and Execution/Resources for Product strategy and execution/PRD Template.rtf`*

## Header

- Title / Initiative
- Date & Version
- Product POC | Design POC | Tech POC | Marketing POC

## Why? (Objective)

- For business
- For users
- How do we measure success?
- Associated OKR/Goal
- Success metrics
- Guardrail metrics (must not harm)

## Who are the users

- Persona
- Problems [Needs, Problems, Jobs to be done]
- How do we know (research evidence)

## Solution

- Brief of the solution
- Alternatives considered with prioritization rationale

## Product flow

- Customer journey
- Wireframes and flow diagrams
- User stories + acceptance criteria
- Edge cases
- Event tracking sheet

| Task | Date |

| Leadership approval | |

| Beta launch | |

## Dependencies

- Open questions
- Infrastructure, budget, partners, internal teams

## Related documents

- Tech planning | Design planning | Go-to-market

## PM workflow (helloPM)

1. Kickoff with discovery (interviews, data, support tickets)
2. Write Why + OKRs — tie to quarterly goals
3. Align on problem before solution
4. Map alternatives and tradeoffs
5. Product flow with journey, wireframes, stories
6. Acceptance criteria per story
7. Tentative dates and dependencies
8. Event tracking plan
9. Share async → iterate → PRD as shared decision artifact
